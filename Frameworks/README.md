# Frameworks del Ecosistema ARMIS

## Ecosistema de Frameworks Especializados de ARMIS

---

Los Frameworks constituyen el núcleo metodológico del ecosistema **ARMIS (Asset Reliability & Maintenance Intelligence System)**.

Cada Framework desarrolla un dominio específico de conocimiento, conservando una filosofía, arquitectura y metodología comunes, definidas en los documentos rectores del proyecto.

Su propósito es transformar los principios establecidos por ARMIS en modelos de ingeniería especializados, capaces de responder a necesidades concretas sin perder la visión integral del ecosistema.

---

# Filosofía de los Frameworks

Cada Framework representa una especialización dentro de un único sistema de ingeniería.

Aunque cada uno aborda un dominio diferente, todos comparten los mismos principios fundamentales:

* Integración con la estrategia organizacional.
* Orientación hacia la generación de valor.
* Gestión basada en procesos críticos y servicios esenciales.
* Pensamiento sistémico.
* Gestión integral de activos, riesgos y tecnología.
* Uso de datos e inteligencia artificial para apoyar la toma de decisiones.
* Mejora continua y evolución permanente.

En consecuencia, los Frameworks no deben entenderse como metodologías independientes, sino como componentes especializados de un mismo ecosistema.

---

# Arquitectura del ecosistema

El ecosistema ARMIS se estructura mediante Frameworks especializados que interactúan de manera coordinada.

```text
ARMIS
│
├── ARMIS-PM
├── ARMIS-AM
├── ARMIS-RM
├── ARMIS-RISK
├── ARMIS-GRC
├── ARMIS-OT
├── ARMIS-AI
└── ARMIS-DA
```

Cada Framework posee documentación propia y puede evolucionar de forma independiente, siempre respetando la Filosofía de Ingeniería ARMIS y el ARMIS Master Plan.

# Relación entre los Frameworks

Cada Framework desarrolla capacidades específicas.

Sin embargo, ninguno constituye una solución aislada.

Todos se complementan para fortalecer la capacidad de las organizaciones de cumplir su misión mediante la integración de estrategia, procesos, servicios esenciales, activos, personas, tecnología e información.

Esta integración constituye uno de los principios fundamentales del ecosistema ARMIS.

---

# Organización documental

Cada Framework seguirá una estructura documental uniforme.

Como mínimo, contará con:

* README.md
* Filosofía específica (cuando aplique).
* Arquitectura.
* Metodología.
* Modelos.
* Herramientas.
* Casos de aplicación.
* Recursos complementarios.

Esta organización garantiza consistencia, trazabilidad y facilidad de navegación dentro del ecosistema.

---

# Estado actual

Actualmente el ecosistema se encuentra en la fase inicial de construcción.

El desarrollo comienza con **ARMIS-PM**, considerado el primer Framework especializado y la base metodológica para la evolución de los demás componentes del ecosistema.

Los Frameworks restantes serán desarrollados progresivamente conforme a la hoja de ruta establecida en el **ARMIS Master Plan**.

---

# Cómo comenzar

Si deseas explorar el ecosistema ARMIS, se recomienda el siguiente recorrido:

1. Leer el README principal del proyecto.
2. Revisar la Filosofía de Ingeniería ARMIS.
3. Consultar el ARMIS Master Plan.
4. Explorar este directorio de Frameworks.
5. Iniciar por **ARMIS-PM**, primer Framework especializado del ecosistema.
